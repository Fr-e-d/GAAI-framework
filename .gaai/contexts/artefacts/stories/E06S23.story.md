# Story E06S23: D1 Edge Serving + Cron Sync

## User Story

**As a** platform operator,
**I want** expert pool data served from D1 at the edge with automatic sync from Supabase,
**so that** matching reads complete in <5ms without hitting the origin database.

---

## Context

Expert pool reads are the hottest read path (every prospect submit + match compute). D1 provides edge SQL with sub-5ms reads. Combined with Cache API L1 (E06S19), the read chain becomes: Cache API (~0ms) → D1 (~1-5ms) → Hyperdrive fallback (~30ms).

A Cron Trigger every 5 minutes incrementally syncs expert data from Supabase (source of truth) to D1 via Hyperdrive. First sync = full load; subsequent syncs = `WHERE updated_at > last_sync_at`.

---

## Acceptance Criteria

- [ ] AC1: D1 database `expert-pool-edge` with binding `EXPERT_DB`
- [ ] AC2: Denormalized `expert_pool` table with scoring-relevant fields
- [ ] AC3: `_sync_meta` table stores `last_sync_at` for incremental sync
- [ ] AC4: Cron `*/5 * * * *` triggers `syncExpertPoolToD1()`
- [ ] AC5: Sync reads from Supabase via Hyperdrive (benefits from connection pooling)
- [ ] AC6: Incremental: `WHERE updated_at > last_sync_at` + lead counts
- [ ] AC7: First sync = full load (no `last_sync_at` → load all)
- [ ] AC8: Read chain: Cache API → D1 → Hyperdrive fallback
- [ ] AC9: Sync failure → Hyperdrive fallback (graceful degradation)
- [ ] AC10: `scheduled()` handler dispatches to sync function
- [ ] AC11: All tests pass

## Out of Scope

- Real-time sync via Durable Objects (E06S25)
- Satellite-specific D1 filtering (post-MVP)
- D1 write path (D1 is read-only in this architecture)

---

## Dependencies

- E06S18 (Hyperdrive + postgres.js for sync reads)
- E06S19 (Cache API L1 in expertPool.ts)
- **Human action required:** Create D1 database via `wrangler d1 create expert-pool-edge`

---

## Notes

D1 schema is denormalized for read performance — no joins needed at query time. The sync function handles both INSERT and UPDATE (upsert pattern). Lead counts are aggregated during sync to avoid per-request count queries.
